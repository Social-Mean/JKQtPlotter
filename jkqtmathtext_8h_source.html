49/*! \brief this class parses a LaTeX string and can then draw the contained text/equation onto a <a href="http://doc.qt.io/qt-5/qpainter.html">QPainter</a>
50 \ingroup jkqtmathtext_render
51
52
53 JKQTMathText is a self-contained LaTeX-renderer for Qt. It is used to renderer
54 labels in JKQTPlotter/JKQTBasePlotter, but can be used independently.
55 The class does not depend on any library, except Qt.
56 In particular it actually parses a LaTeX string and draws it in pure C++. It does NOT rely
57 on an installed LaTeX for the rendering!
58
59 \see See \ref jkqtmathtext_supportedlatex for a description of the supported LaTeX subset
60 and \ref jkqtmathtext_renderingmodel for a description of the rendering model.
61
62
63 \section JKQTMathTextUsage Usage
64
65 \subsection JKQTMathTextUsageDirect Drawing Functions
66 The class provides different variants of the drawing function draw() that paints using an
67 externlly provided <a href="https://doc.qt.io/qt-6/qpainter.html">QPainter</a>. These variants
68 either paint into a QRect, or starting from a single location (x,y).
69
70 The QRect-variants can align the render result inside the rectangle, whereas the
71 location-variants draw from a position that is on the left-hand side of the output's baseline:
72
73 \image html jkqtmathtext/jkqtmathtext_node_geo.png
74
75
76
77 \subsection JKQTMathTextUsageConvenience Convenience Functions
78
79 Alternatively you can use these methods to directly generate a QPixmap or QPicture:
80 - drawIntoPixmap()
81 - drawIntoImage()
82 - drawIntoPicture()
83 .

87
88 In addition there are also functions that allow to calculate the size of the equation,
89 before drawing it (just like the functions in <a href="http://doc.qt.io/qt-6/qfontmetrics.html">QFontMetrics</a>
90 or <a href="http://doc.qt.io/qt-6/qfontmetricsf.html">QFontMetricsF</a>):
91 - getSizeDetail()

104 \subsection JKQTMathTextErrorHandling Error Handling
105
106 The class is designed to be as robust as possible and will still return some output, even if the equation contains some errors.
107 Nevertheless, several errors are detected while parsing. You can get a list of error messages using getErrorList() after calling parse().
108 Also parse() will return \c false if an error occured while parsing.
109
110 \subsection JKQTMathTextUsageExample Example Code
111 This small piece of C++ code may serve as an example of the usage and capabilities of the class:
112 \code
113 // create a JKQTMathText object.
114 JKQTMathText mathText;
115
116 // configure its properties to influence the rendering (e.g. fonts to use, font size, ...)
117 mathText.useXITS();
118 mathText.setFontSize(20);
119
120 // parse some LaTeX code (the Schroedinger's equation)
121 mathText.parse("$\\left[-\\frac{\\hbar^2}{2m}\\frac{\\partial^2}{\\partial x^2}+V(x)\\right]\\Psi(x)=\\mathrm{i}\\hbar\\frac{\\partial}{\\partial t}\\Psi(x)$");
122
123 // draw the result into a QPixmap
124 QPixmap result=mathText.drawIntoPixmap();
125 \endcode
126
127 Alternatively you can also use this class with a <a href="https://doc.qt.io/qt-6/qpainter.html">QPainter</a>:
128
129 \code
130 // use the draw() methods to draw the equation using a QPainter (here onto a QPixmap)
131 QPainter painter;
132
133 // first we determine the size of the render output:
134 const JKQTMathTextNodeSize size=getSizeDetail(painter);
135
136 // now we can us that size information to render:
137 mathText.draw(painter, Qt::AlignCenter, QRectF(0,0,pix.width(), pix.height()), false);
138 \endcode

158 \section JKQTMathTextInternalDetails Implementation Details
159 \subsection JKQTMathTextSuppoertedFonts Font Handling
160
161 Several fonts are defined as properties to the class:
162 - A "roman" (MTEroman / MTEmathRoman) font used as the standard font ( setFontRoman() and for use in math mode setFontMathRoman() )
163 - A "sans-serif" (MTEsans / MTEmathSans) font which may be activated with \c \\sf ... ( setFontSans() and for use in math mode setFontMathSans() )
164 - A "typewriter" (MTEtypewriter) font which may be activated with \c \\tt ... ( setFontTypewriter() )
165 - A "script" (MTEscript) font which may be activated with \c \\script ... ( setFontScript() )
166 - A "math-roman" (MTEmathRoman) font used as the standard font in math mode ( setFontMathRoman() )
167 - A "math-sans-serif" (MTEmathSans) used as sans serif font in math mode ( setFontMathSans() )
168 - A "blackboard" (MTEblackboard) font used to display double stroked characters ( setFontBlackboard() )
169 - A "caligraphic" (MTEcaligraphic) font used to display caligraphic characters ( setFontCaligraphic() )
170 - A "fraktur" (MTEfraktur) font used to display fraktur characters ( setFontFraktur() )
171 - A fallback font MTEFallbackSymbols for (math) symbols, greek letters ... (if the symbols are not present in the currently used font). ( setFallbackFontSymbols() )
172 .
173
174 These fonts are generic font classes, which font is actually used can be configured in JKQTMathText class with the \c set...() functions mentioned above. You can also use these functions to set the fonts used for math rendering in math-mode:
175 - useSTIX() use the STIX fonts from <a href="https://www.stixfonts.org/">https://www.stixfonts.org/</a> in math-mode<br>\image html jkqtmathtext/jkqtmathtext_stix.png
176 - useXITS() use the XITS fonts from <a href="https://github.com/alif-type/xits">https://github.com/alif-type/xits</a> in math-mode. These are included by default in this library and also activated by default.<br>\image html jkqtmathtext/jkqtmathtext_xits.png
177 - useASANA() use the ASANA fonts from <a href="https://ctan.org/tex-archive/fonts/Asana-Math/">https://ctan.org/tex-archive/fonts/Asana-Math/</a> in math-mode<br>\image html jkqtmathtext/jkqtmathtext_asana.png
178 - useAnyUnicode() use generic Unicode fonts, e.g. "Arial" and "Times New Roman" in math-mode. You should use fonts that contain as many of the mathematical symbols as possible to ensure good rendering results.<br>using "Times New Roman": \image html jkqtmathtext/jkqtmathtext_timesnewroman.png
179 <br>using "Arial": \image html jkqtmathtext/jkqtmathtext_arial.png
180 <br>using "Courier New": \image html jkqtmathtext/jkqtmathtext_couriernew.png
181 <br>using "Comic Sans MS": \image html jkqtmathtext/jkqtmathtext_comicsans.png
182 <br>using "Old English Text": \image html jkqtmathtext/jkqtmathtext_OldEnglish.png
183 <br>using "Computer Modern": \image html jkqtmathtext/jkqtmathtext_computermodern.png
184 <br>using "Fira": \image html jkqtmathtext/jkqtmathtext_fira.png
185 <br>using "MS Segoe UI": \image html jkqtmathtext/jkqtmathtext_mssegoeui.png
186 <br>using "DejaVu Sans": \image html jkqtmathtext/jkqtmathtext_DejaVuSans.png
187 <br>using "DejaVu Serif": \image html jkqtmathtext/jkqtmathtext_DejaVuSerif.png
188 .
189
190 Math-mode is activated by enclosing your equation in \c $...$ or \c \\[...\\] . This mode is optimized for mathematical equations. Here is an example of the difference:
191 - <b>math-mode (MTEmathRoman and MTEmathSans, whitespaces are mostly not drawn directly, symbol spacing is different)</b> \c $...$: <br>\image html jkqtmathtext/schreq_mathmode.png
192 - <b>normal mode (MTEroman and MTEsans is used, whitespaces are evaluated directly)</b>: <br>\image html jkqtmathtext/schreq_normalmode.png
193 .
194
195 Font Lookup for symbols works as follows in JKQTMathTextSymbolNode:
196 - if a character is found in the current (or to be used) font, it is taken from there
197 - if the character is not found, it is looked for in the fallback fonts MTEFallbackSymbols
198 - as a last resort, some symbols can be created otherwise, so if neither of the two options above
199 contain the required symbol, the symbol might be synthesized otherwise, or a rectangle with the size of "X" is drawn instead
200 .
